Design and technical comments

Overall structure

Four HTML files (index.html, past.html, future.html, comments.html) and one stylesheet (websystems.css) in the same folder. Images are in websystems/images/. Each page repeats the same header, navigation, and footer so the layout is consistent. Navigation is a semantic list inside a <nav>.

How pages interact

Navigation links connect all pages; the current page uses aria-current="page" and a .current class for styling. If CSS is disabled, the site still works because links are plain text anchors and layout is linear and readable.

Styles and selectors used

Key selectors and why they’re used
Selector Used on Purpose
.nav a Header menu links Horizontal layout, spacing and focus/hover styles for accessibility
.current Active nav link Indicates the current page without relying on colour alone (underline & weight)
#hero (id) Home page hero section Distinct card styling to highlight the introduction
.card Content panels Consistent padding, border radius and subtle shadow
.btn Text links styled as buttons Clear call-to-action while remaining accessible links

Accessibility and robustness

  • Meaningful alt text on images; images include width/height so pages don’t jump while loading.
  • Skip-link for keyboard users; visible focus styles; headings follow a logical order.
  • No external frameworks or imported styles. Works in modern browsers and remains usable with CSS off.

When CSS is turned off

Content reads top-to-bottom with the menu as a simple list. Because there are no background images, all text remains readable. The site avoids layout that depends on CSS to make links or content discoverable.
